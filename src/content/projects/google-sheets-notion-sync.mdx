problem: "A consultancy firm used Google Sheets for client-facing reporting and Notion for internal project tracking. The same data lived in both places, and a team member had to manually copy updates between systems — a task that consumed 2–3 hours every week and regularly resulted in out-of-sync records causing confusion."
solution: "Built a two-way sync pipeline in n8n. Google Sheets changes trigger a webhook that updates the matching Notion database row using the record ID as the key. Notion page updates trigger via polling that pushes changes back to the corresponding Google Sheets row. Added conflict resolution logic to handle simultaneous edits, using timestamp comparison to determine which version is authoritative."
