problem: "A membership-based business was running weekly promotional campaigns via Telegram. The team manually exported member lists, pasted messages one by one, and had no way to track who received what. Campaigns regularly ran late, some members received duplicates, and there was no visibility into delivery failures."
solution: "Built an n8n campaign pipeline triggered by a simple webhook. It fetches the eligible member list from MySQL based on configurable filters (tier, join date, activity), processes them in batches of 50 using SplitInBatches, sends personalised messages via Telegram Bot API with rate limiting to avoid API blocks, logs each delivery result back to MySQL, and runs an automatic retry for any failed sends after a 30-minute delay."
